Version 0.6-0

Improvements to the code generation mechanism.
  • writeCode has a namespace parameter to control whether the NAMESPACE file is written.
    writeNamespace is now exported.
  • writeCode returns information that can be passed to writeNamespace.
  • Processing of the classes and determining the unique names is done in generateInterface() rather than writeCode. This makes dealing with the namespace easier.

    • Version 0.5-0

  • Changes to the C code to deal with changes to changes in R about how strings are stored.
    Enhancements to the code generation mechanism from type library information.
  • Handle CoClass types, enums written as individual variables by default, run time code is now in RDCOMClient, etc.

    • Version 0.4-4

  • Code generation mechanism for creating R code from a type library.
    The resulting code is slightly more efficient and provides much more information at run time about the types and possible methods for a DCOM interface.
  • LoadTypeLib() is now generic with methods for for an ITypeInfo or COMIDispatch object. These ensure that the proper type library is loaded. This should be used in preference to getTypeLib() until the latter is modified.
  • getTypeLib() for an ITypeInfo or COMIDispatch object now returns an object of class IContainingTypeLib. This is slightly different from a regular ITypeLib that is loaded from a file. Note that one can get the complete ITypeLib via LoadTypeLib() on the same object.
  • queryTypeLibPath() returns information about the location of the type library for an ITypeInfo or COMIDispatch object.

    Version 0.4-3

  • getElements() now supports ITypeInfoAlias objects.

    Version 0.4-2

  • ParameterDescription class no longer has an optional slot.
    It is in the ParameterStyle class.
  • getElements("ITypeCoClass") creates objects of derived classes of ITypeInfo, e.g. ITypeInfoDispatch rather than just generic ITypeInfo objects.
  • getElements("ITypeLib") puts the names of the types on the resulting list. These are the human-readable names taken from the names("ITypeLib") method.
  • [[ method for indexing the elements of a ITypeInfoCoClass.
  • lapply and sapply methods to iterate over the elements in an ITypeLib.
  • Name slot for the parameter descriptions objects now filled in.
  • Classes for the different types of ITypeInfo objects.
    ITypeInfoDispatch, ITypeInfoCoclass, ...
  • getElements() is generic and added methods for ITypeLib and the different ITypeInfo* classes above.
  • Added names to the elements returned by getElements() for a coclass ITypeLibEntry.
    These make the results more human readable.
  • Added getTypeLib() generic for character and ITypeLib.
    Implemented method for COMIDispatch instance.
  • Removed the class ITypeLibEntry as it was identical to ITypeInfo.

    • Version 0.4-1

    Binary built for R 1.8.*

    Version 0.4-0

    Support for indexing a type library by the UUID of an entry.

    Version 0.2-1
